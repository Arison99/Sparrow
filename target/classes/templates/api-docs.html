Sparrow MoMo API Documentation

Welcome to the Sparrow Mobile Money API documentation. This API provides comprehensive endpoints for testing and learning how mobile money services work. You can use these endpoints to simulate real-world mobile money operations including user management, transactions, and various financial services.

Base URL

http://localhost:8080

Quick Start

1. Add a User

curl -X POST http://localhost:8080/admin/addUser \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "phone=+1234567890&pin=1234&balance=100.00"

2. Get All Users (JSON)

curl -X GET http://localhost:8080/api/users

3. Send Money

curl -X POST http://localhost:8080/admin/sendMoney \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "fromPhone=+1234567890&toPhone=+0987654321&amount=50.00"

JSON API Endpoints

User Management

GET /api/users

Retrieve all users in JSON format

Example Response:

[
  {"phone": "+1234567890"},
  {"phone": "+0987654321"}
]
POST /api/users/balance

Get balance for a specific user

Parameters:

  • phone (string, required) - User's phone number

Example Response:

{
  "phone": "+1234567890",
  "balance": 100.50
}
POST /api/users/transactions

Get transaction history for a specific user

Parameters:

  • phone (string, required) - User's phone number

Example Response:

{
  "phone": "+1234567890",
  "transactions": [
    {
      "id": "T1642512345678",
      "description": "Sent to +0987654321",
      "amount": -50.00,
      "date": "2025-01-15 10:30:00"
    }
  ]
}

Service Operations

POST /admin/sendMoney

Transfer money between users

Parameters:

  • fromPhone (string, required) - Sender's phone number
  • toPhone (string, required) - Recipient's phone number
  • amount (number, required) - Amount to transfer

Example cURL:

curl -X POST http://localhost:8080/admin/sendMoney \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "fromPhone=+1234567890&toPhone=+0987654321&amount=50.00"
POST /admin/buyAirtime

Purchase airtime for a user

Parameters:

  • phone (string, required) - User's phone number
  • amount (number, required) - Airtime amount
POST /admin/payWithMoMo

Make payment to a merchant

Parameters:

  • phone (string, required) - User's phone number
  • merchant (string, required) - Merchant name
  • amount (number, required) - Payment amount
POST /admin/savingsLoans

Savings and loan operations

Parameters:

  • phone (string, required) - User's phone number
  • action (string, required) - "save" or "loan"
  • amount (number, required) - Amount

XML API Endpoints

User Management

GET /api/users/xml

Retrieve all users in XML format

Example Response:

<users>
  <user><phone>+1234567890</phone></user>
  <user><phone>+0987654321</phone></user>
</users>

Note

XML endpoints follow the same parameter structure as JSON endpoints but return XML responses. All service operations have corresponding XML versions that can be accessed by appending '/xml' to the endpoint path.

Testing with Different Tools

cURL

Add User

curl -X POST http://localhost:8080/admin/addUser \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "phone=+1234567890&pin=1234&balance=100.00"

Get User Balance

curl -X POST http://localhost:8080/api/users/balance \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "phone=+1234567890"

JavaScript (Fetch)

Add User

fetch('http://localhost:8080/admin/addUser', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded'
  },
  body: 'phone=+1234567890&pin=1234&balance=100.00'
})
.then(response => response.text())
.then(data => console.log(data));

Get Users

fetch('http://localhost:8080/api/users')
.then(response => response.json())
.then(users => console.log(users));

Error Handling

Common Error Responses

{
  "error": "User not found"
}

{
  "error": "Insufficient funds"
}

{
  "error": "Invalid amount"
}

HTTP Status Codes

  • 200 - Success
  • 400 - Bad Request (invalid parameters)
  • 404 - Resource not found
  • 500 - Internal server error
